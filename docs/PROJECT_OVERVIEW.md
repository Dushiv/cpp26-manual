## Что мы делаем

Строим **обучающее приложение по стандарту C++26**. Приложение — это *преподаватель*: пользователь проходит модули, читает теорию с примерами, решает проверочные задачи и в конце **понимает каждую тему**. Это не чеклист-трекер, а среда, где контент живёт внутри.

- **Покрытие C++26 — исчерпывающее:** каждая фича объясняется с мотивацией, примерами и упражнениями. Ничего не замалчиваем, включая мелкие и удалённые возможности.
- **Целевой ученик:** уверенный C++14 + знание C++17. Всё ниже этой планки не объясняем (даём внешнюю ссылку).
- **Критерий успеха:** после прохождения ученик уверенно понимает и может применять фичи C++26.

## Зафиксированные решения (не пересматриваем без явной причины)

- **Стек:** веб-приложение, упакованное как **PWA**. React + подсветка кода (Prism.js/highlight.js). Работает в браузере на десктопе и ставится на главный экран Android (офлайн, локальное хранение). Можно без сборочных инструментов (React через CDN).
- **Контент = данные.** Уроки хранятся как структурированный JSON отдельно от кода приложения. Оболочка — маленький «движок», контент пишется и ревьюится независимо.
- **Базовая планка ученика:** C++14 (уверенно) + C++17 (знает). Фоновые врезки покрывают только промежуток C++17 → C++26 и только то, что нужно для понимания конкретной фичи.
- **Самодостаточность:** ученик НЕ на реальной машине и ничего не компилирует. Приложение автономно.
- **Проверка примеров:** каждый пример прогоняется один раз при подготовке контента и сохраняется проверенный ожидаемый вывод, с которым сверяется ученик. Технически это делает Claude напрямую — компилирует и исполняет код через публичный API Compiler Explorer (godbolt.org), без участия человека и без локального тулчейна (см. скилл `compiling-cpp26-examples` и `prototype/run-on-godbolt.js`).
- **Модель упражнений:** автопроверяемые (предскажи вывод, найди ошибку, выбор варианта/«что здесь UB») — часть прохождения. Свободное «напиши сам» — опциональный челлендж с эталонным решением и ссылкой на Compiler Explorer, на прохождение не влияет.
- **Пропуск задач:** разрешён; пропущенное получает статус `Пропущено` (≠ `Выполнено`), копится в «зоне повторения», модуль показывает «N пропущено». Есть опциональный «строгий режим».
- **Mastery-gating:** модуль закрывается по прохождению проверки усвоения (порог-заглушка 80%, не подтверждён). Строится только на автопроверяемых задачах.
- **Compiler Explorer:** в уроке — только опциональные ссылки «попробовать вживую». В челлендже ученик дополнительно может скомпилировать и исполнить свой код прямо в приложении («Запустить»/«Сверить» — см. `docs/superpowers/specs/2026-06-07-runnable-challenge-design.md`); оба варианта остаются опциональными и не входят в прохождение/mastery-gating.

## Факты о C++26 (база)

C++26 утверждён ISO 28.03.2026 (рабочая редакция N5046). Три флагмана: статическая рефлексия (оператор `^^`, сплайс `[: :]`, `<meta>`), контракты (`pre`/`post`/`contract_assert`), `std::execution`. Сквозная тема — безопасность. На апрель 2026 большинство фич поддерживает GCC 16.1; рефлексию — GCC 16 / экспериментальный Clang / EDG.
